## 🚀 WHAT HAPPENS WHEN YOU PUSH TO GITHUB

### Automatic Deployments (Zero-downtime)

**When you push to main branch:**

1. **Render** automatically:
   - Reads `render.yaml`
   - Pulls latest code
   - Runs: `mvn clean package -DskipTests`
   - Deploys new backend
   - Keeps database/Redis up
   - ~3-5 minutes

2. **Vercel** automatically:
   - Detects new push
   - Reads `vercel.json`
   - Runs: `npm run build` in `ordering-frontend/`
   - Deploys new frontend
   - Updates DNS
   - ~1-2 minutes

3. **Result:** Users see new version without downtime

## 🐛 IF SOMETHING GOES WRONG

### Local Docker Won't Start
```bash
# 1. Check Docker is running
docker ps

# 2. Clean start
docker-compose down
docker-compose up -d --build

# 3. Check logs
docker-compose logs backend
```

### Backend can't connect to DB
```bash
# Check database is running
docker-compose ps

# Check database logs
docker-compose logs postgres

# Manually test connection
docker-compose exec postgres psql -U orderapp_user -d ordering_db
```

### Frontend can't reach backend
```bash
# Check backend health
curl http://localhost:8080/api/health

# Check browser console for CORS errors
# Check proxy.conf.json has correct localhost:8080

# For Vercel: verify ANGULAR_APP_API_URL environment variable
```
